# OD战略诊断框架

## 核心理论来源
- 组织发展(OD)理论
- 系统思维
- 三层次问题分析法

## 三层次问题分析模型

### 第一层：业务症状（表象）
特征：
- 用户直接描述的问题
- 通常是量化指标（如"流量下降30%"）
- 容易引起情绪反应

常见示例：
- 流量成本上升
- 转化率下降
- 团队执行力差
- 利润下滑
- 客户投诉增多

### 第二层：系统原因（结构）
特征：
- 导致症状的根本原因
- 涉及流程、结构、机制
- 需要深入分析才能发现

常见类型：
1. **流程问题**：流程设计不合理，导致效率低下
2. **结构问题**：组织架构不匹配业务发展
3. **能力问题**：团队能力跟不上需求
4. **机制问题**：激励机制或考核机制不当
5. **资源问题**：资源配置不合理

### 第三层：心智模式（根源）
特征：
- 创始人的认知和假设
- 通常是无意识的
- 最难改变但影响最深远

常见表现：
1. **成功路径依赖**："过去这样做成功了，所以未来也应该这样做"
2. **认知盲区**：看不到或不愿意看到某些事实
3. **价值观冲突**：个人价值观与业务需求冲突
4. **恐惧驱动**：决策基于恐惧而非机会
5. **身份认同**：将企业等同于自己，无法接受批评

## 诊断流程

### 1. 症状收集
- 让用户描述问题（不要打断）
- 记录关键数据和时间节点
- 观察情绪反应

### 2. 因果链分析（5个为什么）
示例：
- 问题：流量成本上升
- 为什么？因为转化率下降（还需要花更多钱获客）
- 为什么？因为产品竞争力下降
- 为什么？因为竞品推出了更好的功能
- 为什么？因为我们研发投入不足
- 为什么？因为创始人不重视研发

根源：创始人的心智模式（重营销轻产品）

### 3. 系统分析
使用工具：
- 系统循环图
- 因果回路图
- SWOT分析

### 4. 心智模式识别
观察信号：
- 防御性反应
- 归因外部
- 情绪化表达
- 绝对化语言（"必须"、"一定"）

## 干预策略

### 症状层干预（短期）
- 直接解决问题
- 快速见效
- 但可能治标不治本

### 系统层干预（中期）
- 调整流程和结构
- 需要时间和资源
- 效果较持久

### 心智层干预（长期）
- 改变认知和假设
- 最难但需要时最有效
- 需要教练或顾问介入

## 常见陷阱

1. **症状陷阱**：只解决表面问题，不深入根源
2. **快速修复陷阱**：追求短期效果，忽视长期影响
3. **单一因果陷阱**：认为问题只有一个原因
4. **归因陷阱**：将问题归因于外部或他人
5. **解决方案陷阱**：在没有深入理解问题前就提出解决方案

## 成功案例参考

### 案例1：流量成本上升
- 症状：获客成本从50元上升到150元
- 系统原因：产品复购率低，只能不断获取新用户
- 心智模式：创始人认为增长=拉新，忽视用户留存
- 解决方案：短期优化投放，中期提升产品体验，长期改变增长认知

### 案例2：团队执行力差
- 症状：战略总是落不了地
- 系统原因：组织架构不合理，权责不清
- 心智模式：创始人认为执行力=员工态度
- 解决方案：调整组织架构，明确KPI，同时创始人接受管理培训
